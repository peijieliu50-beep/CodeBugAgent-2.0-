# 部署指南：把应用变成"谁都能打开"的公开网址

> 目标：部署到 **Streamlit Community Cloud**（免费），得到一个形如
> `https://你的应用名.streamlit.app` 的固定网址，持续在线、加密码保护、限额防刷。
>
> 全程约 20 分钟。需要你操作的地方都标了 **【你来做】**。

---

## 〇、开始前的准备

- 一个 **GitHub 账号**（已有 ✅）
- 你的 **DeepSeek API 密钥**：`sk-145fc1e1...`（保管好，别发到公开地方）
- 想好两个东西：
  - **访问口令**（发给朋友，例如 `aigc2026`）
  - **每日调用上限**（防止烧钱，建议 `50`）

> ⚠️ 安全底线：API 密钥**永远不要**写进代码或上传到 GitHub。本项目已配置好用
> Streamlit 后台的加密 Secrets 注入，`.gitignore` 也已排除 `.env` 和 `secrets.toml`。

---

## 一、把项目传到 GitHub 【你来做】

### 方式 A：网页上传（最简单，不用装 Git）

1. 打开 https://github.com/new ，登录。
2. 填仓库名，例如 `react-debug-agent`；选 **Public**（Streamlit 免费版需公开仓库）；
   不要勾任何初始化选项；点 **Create repository**。
3. 进入空仓库页面，点 **uploading an existing file**。
4. 打开本机文件夹 `react_debug_agent`，**全选里面的内容**拖进网页。
   - ⚠️ 确认**没有** `.env`、`.streamlit/secrets.toml` 这两个文件被传上去
     （它们含密钥，本项目已用 .gitignore 排除；网页上传时若看到请手动删掉再传）。
5. 底部点 **Commit changes**。完成。

### 方式 B：用 Git 命令（你熟悉命令行的话）

在 `react_debug_agent` 目录下：
```bash
git init
git add .
git commit -m "init: ReAct PyTorch 调试 Agent"
git branch -M main
git remote add origin https://github.com/你的用户名/react-debug-agent.git
git push -u origin main
```
> `.gitignore` 已确保密钥文件不会被提交，可放心 `git add .`。

---

## 二、在 Streamlit Cloud 部署 【你来做】

1. 打开 https://share.streamlit.io ，点 **Continue with GitHub**，授权登录。
2. 点 **Create app** → **Deploy a public app from GitHub**。
3. 填写：
   - **Repository**：选你刚建的 `你的用户名/react-debug-agent`
   - **Branch**：`main`
   - **Main file path**：填 `web/app.py`  ← 关键，别填错
4. 先**别急着点 Deploy**，点下面的 **Advanced settings**。

---

## 三、配置密钥与口令（最关键的一步）【你来做】

在 Advanced settings 的 **Secrets** 框里，**原样粘贴**下面内容，把值改成你自己的：

```toml
DEEPSEEK_API_KEY = "sk-145fc1e1f47b4dfd98dfc2be33511fb8"
ACCESS_PASSWORD = "aigc2026"
DAILY_CALL_LIMIT = "50"
```

- `DEEPSEEK_API_KEY`：你的真实密钥。
- `ACCESS_PASSWORD`：访问口令，发给朋友的就是这个。
- `DAILY_CALL_LIMIT`：每天全站最多调用多少次大模型，`50` 比较稳妥；`0` 表示不限。

填完点 **Save**，再点 **Deploy**。

---

## 四、等待上线

- 平台会自动安装依赖（首次约 3-8 分钟，torch 较大需耐心等）。
- 看到界面出现 **🔒 访问验证** 页面，就说明成功了。
- 你的网址在浏览器地址栏，形如 `https://react-debug-agent-xxxx.streamlit.app`。
  **把这个网址 + 访问口令发给朋友即可，任何人任何地方都能打开。**

## 五、上线后怎么用 / 维护

| 想做的事 | 怎么做 |
|---------|--------|
| 改密钥/口令/限额 | Streamlit Cloud → 你的 app → ⋮ → Settings → Secrets，改完自动重启 |
| 改了代码想更新 | 把新代码 push 到 GitHub，应用会自动重新部署 |
| 暂停/删除应用 | app 列表里 ⋮ → Reboot / Delete |
| 应用"睡眠"了 | 免费版长时间无访问会休眠，下次访问点一下"wake up"即可，几十秒恢复 |

## 六、关于功能的两点说明（提前告诉朋友）

1. **"运行训练验证"在云端受限**：免费云内存小，本项目已设为安全降级——
   Agent 仍能读代码、分析报错、检索知识库、给出修复方案和本地验证方法，
   但不会在云端真的跑 PyTorch 训练（本地运行则完全正常）。
   - 若想让云端也尝试跑（不推荐，可能崩），在 Secrets 里**不要**设 `LOW_MEMORY`。
   - 若想强制跳过训练更稳定，在 Secrets 里加一行：`LOW_MEMORY = "1"`。
2. **调用次数有上限**：到达 `DAILY_CALL_LIMIT` 后当天暂停，第二天自动恢复，防止被刷爆。

---

## 七、常见问题

- **部署失败、日志报内存错误（OOM）**：在 Secrets 加 `LOW_MEMORY = "1"`，重启应用。
- **朋友进去是空白/报错**：确认 `Main file path` 填的是 `web/app.py`。
- **提示"服务端未配置 API 密钥"**：Secrets 里 `DEEPSEEK_API_KEY` 没填对，重填保存。
- **担心密钥泄露**：密钥只在 Streamlit 后台加密存储，访客和 GitHub 上都看不到。
  万一怀疑泄露，去 DeepSeek 后台重置密钥，再到 Secrets 更新即可。
